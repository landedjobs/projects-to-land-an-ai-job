## How to use this

Difficulty: 🟢 beginner · 🟡 intermediate · 🔴 advanced

## For AI Engineer roles

- 🟢 **"Chat with your docs" (RAG)** — upload PDFs, ask questions, get cited answers. Proves: embeddings, retrieval, grounding.
- 🟡 **Eval harness** — take any LLM feature and build an offline + LLM-as-judge eval set. Proves: you think about quality, which seniors love.
- 🟡 **Research agent** — an agent that plans, searches, and synthesizes with tool use. Proves: orchestration, function calling, guardrails.
- 🔴 **Production RAG over a real corpus** — caching, re-ranking, latency/cost dashboard. Proves: system design at scale.

## For AI Product Engineer roles

- 🟢 **AI feature clone** — rebuild one AI feature of a product you love; write up the trade-offs.
- 🟡 **0→1 AI micro-app** — ship a tiny tool end-to-end (auth, UI, LLM, deploy) and get 10 real users.
- 🟡 **Before/after with evals** — improve a feature and *measure* it.

## For GTM Engineer roles

- 🟢 **Lead enrichment pipeline** — scrape + enrich + score leads with an API/LLM.
- 🟡 **Outbound automation** — personalized outreach generated from signals, with a dashboard.
- 🟡 **Clay/automation build** — a documented revenue workflow others can reuse.

## For RAG / Data roles

- 🟡 **Hybrid search engine** — dense + sparse retrieval with a quality comparison.
- 🔴 **Freshness pipeline** — keep an index live as source data changes; show recall metrics.

## What makes a project "count"

- ✅ **Deployed** and clickable (not just a notebook).
- ✅ **Written up** — problem, approach, trade-offs, what you'd do next.
- ✅ **Measured** — even a tiny eval beats "it works."
- ✅ **Specific** — solves a real, narrow problem.
